# Стартовые проектные вопросы

Это не БА-вопросы, но их ответы определяют атрибуцию проекта и попадают в шапку выходного документа. Момент предъявления: PQ-1 и PQ-2 — после питча инициативы (шаг 0.5), с объяснением зачем; PQ-3 — в шаге 3, перед выдачей документа. Не открывать сессию административными вопросами.

Формат: каждый вопрос предъявляется как список опций + возможность свободного ввода («другое: …»). Если среда позволяет интерактивные опции (кнопки) — использовать их; если нет — показать пронумерованный список и попросить выбрать номер или ввести свой вариант.

---

## PQ-1. Юридическое лицо

**Вопрос:** В рамках какого юрлица выполняется проект?

Опции:
1. Альфа-Инвестиции
2. ГоИнвест
3. Оба (Альфа-Инвестиции + ГоИнвест)
4. Другое: {свободный ввод}

**Логика:** ответ определяет набор опций в PQ-2. При ответе «оба» — задать PQ-2 отдельно для каждого юрлица. Ответ фиксируется в шапке документа в поле «Юрлицо».

## PQ-2. Корпоративные KPI / регуляторика

**Вопрос:** На выполнение каких корпоративных KPI или регуляторных требований направлен проект? (можно выбрать несколько)

Опции для **Альфа-Инвестиции**:
1. Чистая прибыль
2. AUM
3. Активные клиенты
4. Регуляторное требование: {указать нормативный источник}
5. Другое: {свободный ввод}

Опции для **ГоИнвест**:
1. AUM
2. Опер-доход core
3. Опер-доход OTC
4. Регуляторное требование: {указать нормативный источник}
5. Другое: {свободный ввод}

**Логика:** множественный выбор. Если выбрана регуляторика — уточнить источник и дедлайн, пометить `[требует валидации: комплаенс]`, если стейкхолдер не назвал точный акт. Ответ фиксируется в шапке в поле «Целевые KPI / регуляторика» и должен согласовываться с блоком 1 банка вопросов (цели и метрики) — при расхождении зафиксировать противоречие.

## PQ-3. Код бюджета PPM и подраздел КР

**Вопрос (двухчастный, задавать последовательно):**
1. Какой код бюджета PPM у проекта?
   - {свободный ввод}
   - «Кода ещё нет / не знаю» → зафиксировать open question с владельцем (обычно руководитель проекта / финконтролёр) и пометить документ как «атрибуция не завершена».
2. К какому подразделу КР (ключевой результат) относится проект?
   - {свободный ввод}
   - «Не определён» → open question аналогично.

**Логика:** оба значения попадают в шапку документа (поля «Код бюджета PPM», «Подраздел КР») и в имя коммита при сохранении в git.

## Правило перехода

К блоку 1 банка вопросов переходить после того, как по PQ-1 и PQ-2 есть либо ответ, либо зафиксированный open question. PQ-3 интервью не блокирует никогда: задаётся перед выдачей, «не знаю» = open question + пометка «атрибуция не завершена».
