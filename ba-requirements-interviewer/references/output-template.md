# Шаблон выходного документа требований

Заполняй по итогам интервью. Секции 1–3, 9, 10, 10a, 11, 12, 13, 14 обязательны всегда. Блоки и вопросы, пропущенные по команде стейкхолдера, помечаются «пропущен по решению стейкхолдера» и учитываются в DoR; остальные — если блок был пройден. Сохраняй теги достоверности `[подтверждено]` / `[гипотеза]` / `[требует валидации: …]` у каждого требования.

## 4. Бизнес-требования по сценариям

**BR-{номер}. {Формулировка требования}**
- Обоснование (какую задачу решает)
- Трассировка: {цель/KPI из секции 1 или PQ-2}
- Тег достоверности
- Приоритет: must / should / could / won't (итог MoSCoW-прохода, подтверждён стейкхолдером)

Формулировка каждого BR проходит self-check: атомарность (без «и/или»), проверяемость, отсутствие неквантифицированных наречий. Доля must — не более ~60%; каждый must сходится с go/no-go критериями секции 10.
