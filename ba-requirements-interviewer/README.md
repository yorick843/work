# BA Requirements Interviewer

ИИ-агент в роли бизнес-аналитика, проводящий структурированное интервью для сбора бизнес-требований в области финтех-инвестиций (брокеридж, ИИС, торговля ценными бумагами, инвест-приложения). На выходе — документ требований (BRD) с оценкой готовности, выданный в git, Confluence или файлом.

## Возможности

- **Три режима старта:** новое требование, продолжение существующего документа, возобновление прерванного интервью с сохранённого черновика.
- **Два темпа:** классическое интервью (по одному вопросу) или бриф-дамп (свободный рассказ стейкхолдера, агент раскладывает по блокам и спрашивает только пробелы).
- **Банк из 65 вопросов** в 10 блоках: от контекста и целей до критериев приёмки, с follow-up паттернами углубления.
- **Практики БА:** карта стейкхолдеров, MoSCoW-приоритизация, трассируемость требований к KPI, критерии качества формулировок, глоссарий, статусная модель документа.
- **Definition of Ready:** скоринг зрелости требований и явный вердикт «готов к передаче в SA» / «требует дозаполнения».
- **Уважение к стейкхолдеру:** команды управления сессией (пропустить / позже / статус / завершить сейчас), право на «не знаю», делегируемые списки вопросов по ролям (комплаенс, архитектор, ИБ, операции).
- **Петля обратной связи:** фидбек стейкхолдера и наблюдения агента сохраняются рядом с BRD — сырьё для ревизии банка вопросов.

## Структура

```
ba-requirements-interviewer/
├── SKILL.md                                # роль, правила интервью, ход работы
├── CHANGELOG.md                            # история версий
└── references/
    ├── question-bank.md                    # 65 вопросов в 10 блоках + follow-up паттерны
    ├── project-context-questions.md        # стартовые вопросы: юрлицо, KPI, PPM/КР
    └── output-template.md                  # шаблон BRD (15 секций + приложения)
```

## Установка

Скилл распространяется папкой или упакованным архивом `.skill` (zip). Поместите папку в каталог пользовательских скиллов вашей среды Claude (например, `/mnt/skills/user/`) либо загрузите `.skill`-архив через интерфейс, если среда его поддерживает.

Упаковка из исходников выполняется скриптом `package_skill` из skill-creator; архив прикладывайте к GitHub Releases, а не коммитьте в ветку.

## Конфигурация

Перед боевым использованием заполните в `SKILL.md` секцию **«Конфигурация git»**: дефолтный репозиторий требований, структура каталогов, имя файла, формат сообщения коммита, политика push. Пока параметры помечены `{TBD}`, агент запрашивает их у пользователя в момент выдачи.
